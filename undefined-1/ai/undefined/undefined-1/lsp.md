# LSP 의 필요성

## Claude Code가 코드 생성할 때 : LSP 거의 필요 없다.

CLAUDE.md 에 명세 있음\
→ 파일 생성/수정\
→ grep으로 충분히 탐색 가능\
→ 새 프로젝트라 코드베이스 작음

새로 만드는 코드는 Claude가 이미 구조 알고 있음. 탐색할 미지의 코드 없음.

***

## LSP가 진짜 필요한 때 : 기존 대형 코드베이스를 Claude가 수정할 때.

레거시 프로젝트 10만 줄\
→ "이 함수 어디서 호출돼?"\
→ grep으로 찾기 애매\
→ LSP 있으면 정확

\+ 사람이 나중에 IDE로 볼 때 — 그건 Claude 무관하게 개발자 경험(DX) 문제다.

***

## 요약

| 상황                         | LSP 필요성                |
| -------------------------- | ---------------------- |
| Claude Code로 새 프로젝트 생성     | 거의 불필요                 |
| Claude Code로 기존 대형 프로젝트 수정 | 있으면 정확도 향상             |
| 사람이 IDE로 프로젝트 탐색           | IDE가 알아서 띄움, 별도 설정 불필요 |



cf. SPARC 명세 → 새 프로젝트 생성 흐름이면 LSP 신경 안 써도 된다.
