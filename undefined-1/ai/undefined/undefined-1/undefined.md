# 세션은 짧게 가져간다

## 짧은 세션 선호 이유

| 문제      | 설명                                  |
| ------- | ----------------------------------- |
| 컨텍스트 비용 | 세션 길수록 토큰 비용 기하급수적 증가               |
| 캐시 만료   | 5분 넘으면 프롬프트 캐시 TTL 만료 → 재로드 비용      |
| 집중도 저하  | 컨텍스트 길면 LLM 초반 내용 희석 (attention 분산) |
| 에러 전파   | 초반 실수가 긴 세션 내내 영향                   |

***

짧게 쪼개는 패턴

세션 1: 설계/계획

세션 2: 구현 A

세션 3: 구현 B

세션 4: 리뷰/테스트



각 세션 시작 시 CLAUDE.md나 메모리로 컨텍스트 복원.

***

## 긴 세션이 유리한 경우

* 탐색적 디버깅 (이전 시도 결과 계속 참조 필요)
* 연속적 판단이 필요한 복잡한 리팩터링

***

원칙

세션 = 하나의 명확한 목표. 목표 달성 → 종료.
