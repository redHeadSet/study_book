# AI

## Claude code

### Best Practice

{% embed url="https://code.claude.com/docs/en/best-practices" %}

> Claude Code(클로드 코드)를 가장 효과적으로 활용하기 위한 핵심 베스트 프랙티스 요약입니다.
>
> ***
>
> ### 1. 명확한 검증 수단 제공 (가장 중요)
>
> * **자가 수정 루프 형성:** Claude가 작업을 마친 후 스스로 확인할 수 있는 **테스트 세트, 빌드 명령어, 린터(Linter)** 등을 프롬프트에 포함하세요.
> * 성공했다는 말만 믿지 말고, 테스트 출력이나 명령어 결과 같은 실질적인 증거(Evidence)를 보여달라고 요구하세요.
>
> ### 2. 탐색 ➔ 계획 ➔ 구현 단계 나누기
>
> * **플랜 모드(`plan mode`) 활용:** 규모가 큰 작업은 곧바로 코딩하게 하지 말고, 먼저 플랜 모드에서 코드를 분석하고 구현 계획을 세우게 하세요.
> * `Ctrl+G`를 눌러 Claude가 만든 계획을 텍스트 에디터에서 직접 수정한 뒤 구현(Default 모드)으로 넘어가면 실패 확률이 극적으로 줄어듭니다.
>
> ### 3. 컨텍스트(Context) 창 무자비하게 관리하기
>
> * **주기적인 초기화:** 대화가 길어지면 Claude의 기억력이 떨어지고 실수가 잦아집니다. 서로 다른 작업을 시작할 때는 반드시 **`/clear`** 명령어로 컨텍스트를 비워주세요.
> * **조사 전용 서브에이전트 사용:** 코드베이스를 넓게 탐색해야 할 때는 메인 대화창 대신 "서브에이전트를 써서 X를 조사해줘"라고 요청하여 메인 창의 토큰 소모를 방지하세요.
>
> ### 4. `CLAUDE.md` 파일 최적화
>
> * **`/init`** 명령어로 프로젝트 루트에 `CLAUDE.md`를 생성하고, 빌드/테스트 명령어와 프로젝트 고유의 스타일 가이드를 적어두세요.
> * **단, 너무 길면 안 됩니다:** Claude가 이미 잘 아는 일반적인 언어 규칙은 과감히 지우고, 가독성 있게 핵심 규칙만 남겨야 지침을 무시하지 않습니다.
>
> ### 5. 피드백 루프는 빠르고 과감하게
>
> * **즉시 개입:** Claude가 엉뚱한 방향으로 가고 있다면 망설이지 말고 **`Esc`** 키를 눌러 동작을 멈추고 방향을 틀어주세요.
> * **되감기 기능:** 접근 방식이 완전히 꼬였다면 **`Esc + Esc`** 또는 `/rewind`를 통해 이전 체크포인트(코드 및 대화 상태)로 되돌릴 수 있습니다. 동일한 실수를 2번 이상 반복하면 창을 비우고 프롬프트를 수정해 새로 시작하는 것이 좋습니다.
